# Firestoreコレクション追加時のルール

Firestoreに関する新しい機能やコレクションを追加する際は、以下のステップを必ず実行すること。
特に、ステップ4の「firestore.rules の更新」は最も重要であり、これを怠るとデータの保存に失敗する。

1. **型の定義**
   - 扱うデータの `interface` または `type` を定義する。

2. **コレクション参照の定義**
   - `collection(db, "feature_name")` として参照を定義する。

3. **CRUDと購読の実装**
   - データの作成・更新・削除、およびリアルタイム購読を実装する。

4. **【最重要】firestore.rules へのセキュリティルール追記**
   - `firestore.rules` を必ず更新し、追加したコレクションへのアクセス権限（`allow create`, `allow read` 等）を明記する。
   - ルールが欠落していると、クライアントからの書き込みが全てFirestore側で拒否され、コレクションが作成されない原因となる。
