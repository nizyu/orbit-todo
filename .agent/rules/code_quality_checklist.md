# コード品質チェックリスト (Code Quality Checklist)

このプロジェクトでは、コードの品質と一貫性を保つために **Biome** を使用しています。また、コードの信頼性を担保するための**自動テスト**も導入されます。
作業を完了する（Pull Requestの作成や、タスク完了を宣言する）前に、必ず以下の項目を確認してください。

## 必須チェック項目

- [ ] **コードのフォーマットとリントチェック**
  ```bash
  npm run check
  ```
  このコマンドは `biome check --write .` を実行し、フォーマットの適用と安全なリントエラーの自動修正を行います。

- [ ] **テストの追加と更新**
  コードの追加や修正を行った場合は、その振る舞いを検証する**テストコード（ユニットテスト/E2Eテスト等）を必ず追加または更新**してください。

- [ ] **テストの実行**
  （テスト環境構築後）テストを実行し、すべてのテストがパスすることを確認してください。
  ```bash
  npm run test
  ```

### 各種コマンドのリファレンス

- `npm run format`: コードのフォーマットのみを実行します (`biome format --write .`)
- `npm run lint`: リンターによるチェックと自動修正のみを実行します (`biome lint --write .`)
- `npm run check`: 上記の両方を一度に実行します。通常はこちらを使用してください。
- `npm run test`: プロジェクトのテストスイートを実行します。

> [!NOTE]
> 自動修正できないエラーが発生した場合は、手動でコードを修正し、再度 `npm run check` を実行してエラーが解消されたことを確認してください。
> テストが失敗した場合は、修正内容が要件を満たしているか、または過去の仕様を破壊していないかを確認してください。
